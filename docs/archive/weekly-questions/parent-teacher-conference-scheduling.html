Parent-Teacher Conference Scheduling

Recently, the students of Sir Winston Churchill Secondary have been signing up for parent-teacher conferences. When signing up, students provide a preferred start time for their conference, in addition to a duration in minutes. End times are inclusive, so if the start time was 555 minutes after midnight and the duration was 666 the conference would still be going on at minute 111111.

You are to write a program that processes sign-ups. More specifically, given a list of sign-up information in chronological order, you should determine, for each sign-up, whether it conflicts with a previously successfully processed sign-up.

Input Specification​

  • The first line will contain an integer NNN, denoting the number of sign-ups to process.
  • The next NNN lines will each contain an integer TiT_iTi​ indicating the start time of the conference as minutes since midnight and an integer DiD_iDi​ indicating the duration of the conference in minutes.

Output Specification​

Output NNN lines, where each line corresponds to a sign-up in the order that they were given. Each line should contain either GOOD, if the corresponding sign-up does not conflict with any conference scheduled previously, or BAD otherwise.

Examples

Example 1​

Input​

1
1 2

Output​

GOOD

Explanation​

There is only one conference scheduled.

Example 2​

Input​

2
1 5
2 4

Output​

GOOD
BAD

Explanation​

The first conferences runs from minute 111 to 666, while the second runs from 222 to 666, which conflicts with the first.

Example 3​

Input​

3
34 7
1 2
2 35

Output​

GOOD
GOOD
BAD

Explanation​

The first conference runs from minute 343434 to 414141. The second runs from minute 111 to 333. Finally, the third runs from 222 to 353535. However, minute 222 is reserved for the second conference.

Example 4​

Input​

4
1 4
2 1
2 7
6 1

Output​

BAD
BAD
GOOD

Explanation​

The first conference runs from minute 111 to 555. The second runs from minute 222 to 333, and thus conflicts with the first. The third runs from minute 222 to 999, and also conflicts with the first. The last one runs from minute 666 to 777 which does not conflict with any successfully processed sign-up. Note that as the second and third sign-ups conflicted with the first they are not considered processed.

Intuition​

We first consider a subproblem: given exactly two conferences taking place between [a1,b1][a_1, b_1][a1​,b1​] and [a2,b2][a_2, b_2][a2​,b2​], is there any overlap? For example, [1,5][1, 5][1,5] and [3,6][3, 6][3,6] overlap; [2,7][2, 7][2,7] and [8,9][8, 9][8,9] don't.

This problem reduces to checking whether the intersection of the two intervals [a1,b1][a_1, b_1][a1​,b1​] and [a2,b2][a_2, b_2][a2​,b2​] is non-empty, which can be solved using a bit of simple math.

(The method presented is not the only way to do it, but it is probably the shortest.)

In particular, the intersection of two intervals [a1,b1][a_1, b_1][a1​,b1​] and [a2,b2][a_2, b_2][a2​,b2​] is given by:

max⁡(min⁡(b1,b2)−max⁡(a1,a2)+1,0)\max(\min(b_1, b_2) - \max(a_1, a_2) + 1, 0)max(min(b1​,b2​)−max(a1​,a2​)+1,0)

For example, consider the example of [1,5][1, 5][1,5] and [3,6][3, 6][3,6]. We plug in these values to the equation:

max⁡(min⁡(b1,b2)−max⁡(a1,a2)+1,0)=max⁡(min⁡(5,6)−max⁡(1,3)+1,0)=max⁡(5−3+1,0)=max⁡(3,0)=3\max(\min(b_1, b_2) - \max(a_1, a_2) + 1, 0)\\ = \max(\min(5, 6) - \max(1, 3) + 1, 0)\\ = \max(5 - 3 + 1, 0)\\ = \max(3, 0)\\ = 3max(min(b1​,b2​)−max(a1​,a2​)+1,0)=max(min(5,6)−max(1,3)+1,0)=max(5−3+1,0)=max(3,0)=3

...and 333 is indeed the size of the intersection of [1,5][1, 5][1,5] and [3,6][3, 6][3,6], which is [3,5][3, 5][3,5].

After solving this subproblem the larger problem becomes easy; all one needs to do is to maintain a list of all previously processed conference times. Upon receiving a new time, go through this list and see if any of them overlap. If so, it is bad; otherwise, it is good and should be marked as processed.

Code​

def cmp_intersection(i0, i1):
    return max(min(i1[1], i0[1]) - max(i1[0], i0[0]) + 1, 0)

processed = []
n = int(input())
for _ in range(n):
    start, dur = map(int, input().split())
    cur_time = (start, start + dur)
    if any(cmp_intersection(prev_time, cur_time) > 0 for prev_time in processed):
        print("BAD")
    else:
        print("GOOD")
        processed.append(cur_time)
